# Why Less Code Wins

Every line you don't write is a line that can't break.

Good code is not about clever abstractions. It's about solving the problem with the minimum moving parts.

## Principles

- Delete more than you add
- Prefer flat over nested
- Favor convention over configuration

Less code → less surface area → fewer bugs → faster changes.
